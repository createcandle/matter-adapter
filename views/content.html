Matter is a dubious new smart home standard backed by companies like Google.

The way the protocol is designed makes it very hard to control how data flows. While it allows for local network control on the one hand, it also makes is very hard to prevent data from flowing to the internet.

Candle tries to contain and wall-off your Matter network, but Matter's design makes this needlessly hard and fragile. Containment is only possible if you follow these rules:

  • Only connect Matter devices to the Candle controller directly, by clicking on the button below. Do not pair it via another app such as, for example, Apple's Homekit on your iPhone.
  • You should use Candle's built-on WiFi hotspot functionality, and only connect Matter devices to that hotspot.
  • Do not pair a Matter device to both the Candle Matter network and another "border router" device that's directly connected to your router, such as Appple's Homepod. As soon as you do, that's like punching a hole in the wall, and your data will flow out through that border router.
  • Beware Trojans bearing gifts.

If you want to literally keep things simple and avoid corporate surveillance, then use Zigbee whenever possible.

Extending the Thread network

							
Thread is a mesh-network, which means that some nodes (that do not rely on battery power) can relay messages for other nodes. This makes it easy to expand the distance that the network can cover.

							
							
If you have multiple Candle Controllers in your home (for example to enable voice control in different rooms), then each Candle Controller can also help expand the each of the Thread network.

							
To do this, all controllers on the Mesh network need to use the same 'network code'. This code contains details about the network, including the password.
